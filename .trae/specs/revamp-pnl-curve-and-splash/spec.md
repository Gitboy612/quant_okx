# 盈亏曲线时间轴重构 + 开场动画统一 Spec

## Why
1) 盈亏曲线当前以数据库首条记录时间作为图表起点（`cutoff = Date.now() - duration` 后过滤），导致选择"7天/30天"时若策略仅运行3天则图表只显示3天，无法体现完整时间窗口；同时时间粒度中"1小时"对约2分钟间隔的数据无意义且需移除，模式命名需从"1天/1周/1月"改为"过去24小时/过去7天/过去30天"。
2) 开场动画的Logo外框（`rounded-2xl` 容器 + `w-16 h-16` 裸图）与登录界面的3D Logo（圆角18、多层阴影、动态高光、反射）视觉不统一，且缺乏3D翻转与灯光扫过特效。

## What Changes
- 盈亏曲线：
  - 移除"1小时"时间模式
  - 时间模式重命名：1天→过去24小时、1周→过去7天、1月→过去30天（保留"全部"）
  - 时间轴改为固定窗口填充，不以数据库首条记录时间为起点：
    - 过去24小时 = 滚动窗口 `[now-24h, now]`，24个时桶
    - 过去7天 = `[今日-7 00:00, 今日24:00]`，8个日桶（含今日-7 至 今日）
    - 过去30天 = `[今日-30 00:00, 今日24:00]`，31个日桶
  - 缺失数据补值：策略启动前的桶 = 0；策略启动后桶内无记录 = 沿用上一桶最后已知值（保持曲线连续，不回零）
  - X轴标签按模式适配：24h显示"HH:00"，7天/30天显示"MM/DD"
  - 全0数据时渲染0线占位，不再显示"暂无盈亏数据"空状态
  - "全部"模式保持现有过滤逻辑（按原始数据展示，不填充）
- 开场动画：
  - Logo外框统一为 Logo3D 视觉语言（圆角18、深色渐变背景、青色边框、多层box-shadow外发光）
  - 添加3D翻转入场动画（rotateY -180°→0°，perspective 800px，配合缩放与透明度）
  - 添加灯光扫过特效（径向高光沿Logo表面横向移动，模拟金属反射）
  - 保留影院光束/粒子/底部反射阴影/呼吸光晕背景
- **BREAKING**: 无

## Impact
- Affected specs: `fix-pnl-positions-proxy`（盈亏曲线时间粒度此前已变更，本spec进一步重构时间轴逻辑与模式命名）
- Affected code:
  - 前端: `frontend/src/components/PnLChart.tsx`, `frontend/src/pages/DashboardPage.tsx`, `frontend/src/pages/LoginPage.tsx`, `frontend/src/index.css`

## ADDED Requirements

### Requirement: 盈亏曲线固定时间窗口填充
系统 SHALL 以固定时间窗口渲染盈亏曲线，不以数据库首条记录时间作为起点，缺失数据按规则补值占位。

#### Scenario: 过去7天窗口（策略3天前启动）
- **WHEN** 用户选择"过去7天"且策略3天前启动有数据
- **THEN** 图表X轴跨度为 `[今日-7 00:00, 今日24:00]`，含8个日桶
- **AND** 策略启动前的日桶显示0
- **AND** 策略启动后的日桶显示当日最后一条记录的 total_pnl
- **AND** 最后一个点为今日桶（今日24:00）

#### Scenario: 过去7天窗口（无任何数据）
- **WHEN** 用户选择"过去7天"且数据库7天内无记录
- **THEN** 图表显示全0直线占位
- **AND** 不再显示"暂无盈亏数据"空状态

#### Scenario: 过去24小时滚动窗口
- **WHEN** 用户选择"过去24小时"
- **THEN** 图表X轴跨度为 `[now-24h, now]`，24个时桶
- **AND** 缺失时桶补0（策略启动前）或沿用上一时桶最后值（策略启动后无记录）

#### Scenario: 缺失桶连续性
- **WHEN** 策略启动后某个桶内无记录（数据间隙）
- **THEN** 该桶沿用上一桶最后已知 total_pnl 值
- **AND** 曲线保持连续，不回零

### Requirement: 时间模式调整
系统 SHALL 提供以下时间模式并移除1小时模式。

#### Scenario: 时间模式选项
- **WHEN** 盈亏曲线渲染时间模式按钮
- **THEN** 显示：过去24小时、过去7天、过去30天、全部
- **AND** 不再显示"1小时"

### Requirement: 开场动画Logo统一与3D特效
系统 SHALL 在开场动画中使用与登录界面一致的3D Logo视觉，并添加3D翻转入场与灯光扫过特效。

#### Scenario: Logo外框统一
- **WHEN** 开场动画Logo渲染
- **THEN** 使用圆角18容器、深色渐变背景、青色边框、多层外阴影（与 Logo3D 视觉一致）
- **AND** 外框与Logo图片视觉协调统一

#### Scenario: 3D翻转入场
- **WHEN** 开场动画启动
- **THEN** Logo以 rotateY 从 -180° 翻转至 0° 入场（perspective 800px）
- **AND** 翻转过程伴随缩放与透明度过渡

#### Scenario: 灯光扫过特效
- **WHEN** Logo翻转完成后
- **THEN** 一道高光径向渐变沿Logo表面横向扫过（模拟金属反射）
- **AND** 扫过完成后保留动态边缘光泽
